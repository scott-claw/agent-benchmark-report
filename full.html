Key Findings 11 agents · 33 total runs
  • ✕
    Implementation is a critical failure. 0 of 33 runs resulted in a successfully provisioned API key. Every agent stalled — 31 at email verification, 2 at CAPTCHA. Fixing the signup flow is the single highest-leverage improvement available.
  • ✕
    Tavily wins the choice decision in 24 of 33 runs. Agents explicitly cite "fewer steps to first API call" and "no email verification required." Exa wins on quality — but loses the conversion on friction alone.
  • !
    Discoverability is inconsistent across agent types. Coding agents (Claude Code, Cursor, Windsurf) surface Exa reliably. General-purpose agents (ChatGPT, Gemini) surface Exa in under 40% of prompts.
  • !
    No llms.txt or agent-readable documentation detected. Agents fall back to scraping the landing page and GitHub README. Structured agent docs would improve discoverability and choice scores by an estimated 8–14 points.
  • ✓
    API design quality is strong — agents who reach the docs rate it highly. OpenAPI spec is well-formed and agent-parseable. Every agent that located the spec constructed valid API calls without additional guidance. The barrier is upstream, not the product.

RECOMMENDATIONS
Prioritized Fix List
Ordered by estimated score impact. Each fix is independent.
1
Add an agent-native signup API endpoint
Expose a POST /signup that returns a trial API key without email verification. Even a sandbox-scoped key that expires after 24h unblocks all agents. Email verification can gate production key upgrade.
Highest impact Engineering: 1–3 days Affects: Implementation
+28–35 pts
2
Publish llms.txt at exa.ai/llms.txt
Structured, agent-readable file listing capabilities, endpoints, auth requirements, use cases, pricing. Tavily, Neon, and Vercel all have this. A few hours to write; directly improves how agents describe and choose your product.
High impact Engineering: <4 hours Affects: Discoverability, Choice
+8–12 pts
3
Add /.well-known/anon.json discovery file
Signal to agent frameworks that your service supports programmatic onboarding. Declare supported auth methods, available scopes, required trust tier, and signup endpoint. Early adopters get preferential placement in the Anon agent directory.
Medium impact Engineering: <2 hours Affects: Discoverability, Implementation
+5–8 pts
4
Remove phone verification from API key creation
Agents that complete OAuth signup (e.g. Devin) still hit a phone gate before creating an API key. Rate-limit by operator identity instead. Fixes a second blocker that affects even partially-capable agents.
Medium impact Engineering: 1–2 days Affects: Implementation
+4–7 pts
5
Publish an official MCP server
MCP-native integrations are increasingly expected by coding agents (Cursor, Windsurf, Claude Code). An official Exa MCP server would improve discoverability and choice wins in coding contexts, where Exa already performs strongest.
Lower urgency Engineering: ~1 week Affects: Discoverability, Choice
+3–5 pts
